# 型安全イベントシステム リファレンス

> 本文書は `使い方/TypeSafeEventImplementationReport.md` から移設したリファレンスです。

## 概要

SengenUIのイベントシステムは `EventTypes.ts` で定義された型マッピングにより、コンパイル時にイベント名とハンドラ型の整合性を保証する。ランタイムオーバーヘッドはゼロ。

## 型安全なイベント登録

```typescript
// addTypedEventListener — 型推論でeventの型が確定する
button.addTypedEventListener("click", (event) => {
    // event: MouseEvent として型付け済み
    console.log(event.clientX);
});

// 便利メソッド — 頻出イベント用
button.onClick((event) => { ... });      // MouseEvent
input.onFocus((event) => { ... });       // FocusEvent
input.onEnterKey((event) => { ... });    // KeyboardEvent
```

## イベントハンドラ型マッピング

`EventHandlerMap` インターフェースにより、イベント名 → ハンドラ型の対応をコンパイル時に検証する。

```typescript
// 型エラーになる例
button.addTypedEventListener("click", (event: KeyboardEvent) => { ... });
// Error: KeyboardEvent は MouseEvent に割り当てられない
```

## 注意事項
- `addEventListener` は使わない。必ず `addTypedEventListener` またはコンビニエンスメソッドを使う。
- リスナー削除は `removeTypedEventListener` で行う。
